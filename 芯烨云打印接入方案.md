# 芯烨云打印接入方案

> 整理日期：2026-06-09  
> 适用项目：停云山居小程序  
> 目标场景：扫码点餐后厨提示并出票；新预定订单通知出票

## 结论

芯烨云打印机可以用于本项目。

它的开放平台提供“小票订单打印”接口，支持把订单内容发送到云打印机，并支持来单语音提示、离线队列、打印状态查询和状态回调等能力。适合用于：

- 扫码点餐下单后，后厨自动响铃/语音提示并打印小票。
- 顾客提交用餐预定、住宿预定后，前台或后厨自动打印“新预定通知单”。
- 后台记录打印成功、失败、离线等状态，便于人工补打。

官方文档地址：https://www.xpyun.net/open/index.html

## 不建议的接入方式

不要在微信小程序前端直接调用芯烨云接口。

原因是芯烨云接口需要开发者 ID 和开发者密钥 `UserKEY` 生成签名。如果把密钥放在小程序端，发布后有泄露风险，别人可能拿到密钥后伪造打印请求。

本地目录 `xpyun-opensdk-miniprogram-demo/` 可以作为接口调用和小票模板参考，但不建议生产环境照搬它的小程序端直连方式。

## 推荐接入方式

使用云函数/服务端调用芯烨云接口。

推荐流程：

```text
用户扫码点餐或提交预定
  -> 小程序调用现有云函数
  -> 云函数创建业务订单
  -> 云函数生成小票内容
  -> 云函数调用芯烨云打印接口
  -> 保存打印订单号和打印状态
```

需要把以下配置放在云函数环境变量或安全配置里：

- `XPYUN_USER`：芯烨云开发者 ID
- `XPYUN_USER_KEY`：芯烨云开发者密钥
- `XPYUN_PRINTER_SN`：打印机编号，通常在机器底部 SN/PID 标签上

## 项目接入点

### 1. 扫码点餐出票

文件位置：

```text
tingyun2026/cloudfunctions/mealOrderManage/index.js
```

建议在 `createMealOrder()` 中，订单写入 `meal_orders` 成功后调用云打印。

打印内容建议包含：

- 标题：停云山居厨房单
- 订单号
- 桌号/桌名
- 用餐人数
- 首单或加菜单
- 菜品名称、数量、单价、小计
- 快捷备注和用户备注
- 下单时间

建议参数：

```js
{
  voice: 2,        // 来单播放模式
  copies: 1,       // 打印 1 份
  mode: 1,         // 不检查在线状态，离线时进入队列
  idempotent: orderNo
}
```

`idempotent` 用订单号做幂等标识，避免短时间重复提交导致重复打印。

### 2. 新预定订单出票

文件位置：

```text
tingyun2026/cloudfunctions/reservationManage/index.js
```

建议接入位置：

- `createDiningReservation()`：创建用餐预定后打印“新用餐预定通知单”。
- `createAccommodationReservation()`：创建住宿预定后打印“新住宿预定通知单”。
- `simulateWechatPay()` 或真实支付回调：散客支付成功后打印正式通知单。

预定通知单建议包含：

- 标题：新预定通知
- 订单号
- 预定类型：用餐/住宿
- 联系人和手机号
- 日期、时段或入住离店日期
- 人数
- 包间/房间
- 餐标或住宿金额
- 备注
- 当前状态：待支付、已支付待确认、会员待线下核对等

## 打印接口要点

芯烨云小票打印接口：

```text
POST https://open.xpyun.net/api/openapi/xprinter/print
Content-Type: application/json;charset=UTF-8
```

公共参数：

- `user`：芯烨云开发者 ID
- `timestamp`：10 位 Unix 时间戳，单位秒
- `sign`：`user + UserKEY + timestamp` 拼接后做 SHA1

打印参数：

- `sn`：打印机编号
- `content`：打印内容，不能超过 12K
- `copies`：打印份数
- `voice`：语音模式，`2` 为来单播放模式
- `mode`：打印模式，`1` 表示离线时也创建打印订单并进入队列
- `idempotent`：唯一订单号标识，用于去重

## 小票模板标签

芯烨云小票内容支持简单标签排版：

```text
<BR> 换行
<L></L> 左对齐
<C></C> 居中
<R></R> 右对齐
<B></B> 放大
<BOLD></BOLD> 加粗
<CUT> 切刀
<QRCODE>二维码内容</QRCODE>
```

示例：

```text
<C><B>停云山居厨房单</B><BR></C>
<L>订单号：TYMEAL20260609123000ABCD<BR></L>
<L>桌号：A01    人数：4<BR></L>
--------------------------------<BR>
<L>竹笋炒肉 x2<BR></L>
<L>清炒时蔬 x1<BR></L>
--------------------------------<BR>
<L>备注：少辣<BR></L>
<CUT>
```

## 到货后测试清单

1. 注册芯烨云开发者账号。
2. 登录芯烨云后台，获取开发者 ID 和 `UserKEY`。
3. 给打印机通电、联网。
4. 在芯烨云后台添加打印机 SN/PID。
5. 在后台确认打印机状态为在线。
6. 先用芯烨云后台或 demo 打印一张测试小票。
7. 确认声音提示是否满足后厨环境。
8. 确认纸宽：58mm 或 80mm。
9. 确认是否需要自动切纸。
10. 确认需要打印几份：厨房 1 份、前台 1 份，或只打印厨房单。

## 开发实施计划

### 第一步：封装打印服务

在云函数中新增统一打印模块，建议路径：

```text
tingyun2026/cloudfunctions/shared/xpyun-printer.js
```

负责：

- 读取环境变量。
- 生成 SHA1 签名。
- 调用芯烨云接口。
- 统一处理返回码。
- 生成点餐小票和预定通知单模板。

### 第二步：点餐下单后打印

在 `mealOrderManage/createMealOrder()` 里：

- 创建订单成功。
- 调用打印服务。
- 把返回的芯烨云打印订单号写入 `meal_orders`。
- 打印失败时不要影响用户下单，但要记录 `print_status = failed` 和错误信息。

### 第三步：预定订单打印

在 `reservationManage` 里：

- 会员预定：提交后可立即打印。
- 散客预定：建议支付成功后打印。
- 如果只是内部提醒“有新预定待支付”，也可以创建时打印一张“待支付预定提醒单”。

### 第四步：后台补打和状态查询

后续后台管理可以增加：

- 查看打印状态。
- 手动补打。
- 查询打印机在线状态。
- 查询某张小票是否打印成功。
- 打印失败提醒。

## 需要你购买前确认

购买时建议确认这些信息：

- 支持“芯烨云”开放平台。
- 是云打印机，不只是蓝牙/USB 打印机。
- 支持小票打印，适合餐饮后厨。
- 纸宽建议 80mm，厨房看单更清楚。
- 后厨噪声较大时，确认来单语音/蜂鸣音够明显。
- 如果需要自动切纸，购买支持切刀的型号。
- 网络方式确认：Wi-Fi、4G 或网线。

## 暂定决策

当前先购买机器并完成基础开通。机器到货后再继续：

1. 获取并填写芯烨云账号、密钥、打印机 SN。
2. 用 demo 或云函数打印测试单。
3. 接入点餐小票。
4. 接入预定通知单。
5. 增加打印状态记录和补打能力。
