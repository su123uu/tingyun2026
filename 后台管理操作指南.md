## 后台架构

后台分两层权限：

1. CloudBase 身份认证层  
   前端会使用 CloudBase Web SDK 做匿名登录。匿名登录只是为了让网页端有资格调用指定云函数。

2. 后台管理员层  
   `adminManage` 云函数内部校验管理员账号密码。只有登录成功后，前端才拿到后台 token，并继续调用增删改查接口。

目前没有直接使用 CloudBase 的“用户名密码用户体系”作为后台管理员体系。后续可以升级为：CloudBase 用户名密码登录 + 管理员 UID 白名单。

## 云函数权限规则

当前云函数安全规则是只开放 `adminManage` 给已登录用户调用，其它云函数仍禁止匿名用户调用：

```json
{
  "*": {
    "invoke": "auth != null && auth.loginType != 'ANONYMOUS'"
  },
  "adminManage": {
    "invoke": "auth != null"
  }
}
```

这样做的含义：

- Vue 后台匿名登录后可以调用 `adminManage`。
- 其它云函数仍不允许匿名用户调用。
- 真正的后台操作权限仍由 `adminManage` 内部的管理员密码校验控制。

## adminManage 环境变量

`adminManage` 云函数需要以下环境变量：

```text
ADMIN_USERNAME=admin
ADMIN_API_TOKEN=tingyun2026bannerAdmin
bannerAdmin=tingyun2026bannerAdmin
```

其中：

- `ADMIN_USERNAME` 是后台登录账号。
- `ADMIN_API_TOKEN` 是后台登录密码和后续管理 token。
- `bannerAdmin` 是兼容旧逻辑的备用变量。

## 部署 Vue 后台

推荐用静态方式部署 `tysj8/dist`，不要让 CloudBase 云端重新跑 Vite 构建。

临时创建一个部署配置文件，例如项目根目录的 `tmp-static-app-cloudbaserc.json`：

```json
{
  "version": "2.0",
  "envId": "cloud1-d6gzs6wuu4b4e902e",
  "app": {
    "serviceName": "tysj8",
    "framework": "static",
    "root": "./tysj8/dist",
    "installCommand": "",
    "buildCommand": "",
    "outputDir": ".",
    "deployPath": "/tysj8"
  }
}
```

部署命令：

```powershell
tcb app deploy tysj8 -e cloud1-d6gzs6wuu4b4e902e --config-file .\tmp-static-app-cloudbaserc.json -f --json
```

部署成功后检查：

```powershell
tcb app info tysj8 -e cloud1-d6gzs6wuu4b4e902e --json
```

确认 `latestStatus` 是：

```text
SUCCESS
```

部署完成后可以删除临时配置文件。

## 部署 adminManage 云函数

临时创建一个云函数部署配置，例如项目根目录的 `tmp-adminmanage-cloudbaserc.json`：

```json
{
  "envId": "cloud1-d6gzs6wuu4b4e902e",
  "functionRoot": "./tingyun2026/cloudfunctions",
  "functions": [
    {
      "name": "adminManage",
      "runtime": "Nodejs16.13",
      "handler": "index.main",
      "timeout": 10,
      "memorySize": 256,
      "envVariables": {
        "ADMIN_USERNAME": "admin",
        "ADMIN_API_TOKEN": "tingyun2026bannerAdmin",
        "bannerAdmin": "tingyun2026bannerAdmin"
      }
    }
  ]
}
```

部署命令：

```powershell
tcb fn deploy adminManage --force --config-file .\tmp-adminmanage-cloudbaserc.json --json
```

部署后检查云函数详情：

```powershell
tcb fn detail adminManage -e cloud1-d6gzs6wuu4b4e902e --json
```

部署完成后可以删除临时配置文件。

## 验证登录接口

由于 Windows PowerShell 直接传 JSON 给 `tcb fn invoke --params` 容易转义出错，推荐新会话里用 Node 脚本或让 AI 直接用参数数组方式验证。

验证请求内容：

```json
{
  "action": "login",
  "username": "admin",
  "password": "tingyun2026bannerAdmin"
}
```

成功返回类似：

```json
{
  "ok": true,
  "data": {
    "username": "admin",
    "token": "tingyun2026bannerAdmin"
  }
}
```

## 常见问题

### 打开后台后仍然直接进管理页

大概率是浏览器缓存旧版本。

处理：

```text
Ctrl + F5 强制刷新
```

或者打开无痕窗口访问后台地址。

### 提示 PERMISSION_DENIED

优先检查云函数安全规则：

```powershell
tcb permission get function -e cloud1-d6gzs6wuu4b4e902e --json
```

规则里必须包含：

```json
"adminManage": {
  "invoke": "auth != null"
}
```

如果没有，网页匿名登录后无法调用 `adminManage`。

### 提示请开启匿名登录

进入腾讯云 CloudBase 控制台：

```text
身份认证 -> 登录方式 / 身份源 -> 开启匿名登录
```

匿名登录是为了让网页能调用 `adminManage`，不是后台管理员登录。

### 登录提示账号或密码不正确

检查 `adminManage` 云函数环境变量：

```powershell
tcb fn detail adminManage -e cloud1-d6gzs6wuu4b4e902e --json
```

确认有：

```text
ADMIN_USERNAME=admin
ADMIN_API_TOKEN=tingyun2026bannerAdmin
```

### 云应用状态失败

不要优先使用 `framework: vue` 或 `framework: vite` 的云端构建。当前更稳的方式是：

1. 本地执行 `npm run build`
2. 用 `framework: static` 部署 `tysj8/dist`

### 本地预览打不开

先确认本地服务是否真的启动：

```powershell
Invoke-WebRequest -Uri 'http://127.0.0.1:5173/' -UseBasicParsing
```

能返回 `200` 才说明本地预览服务正常。

如果 `npm run dev` 后浏览器打不开，优先用：

```powershell
cd E:\AI编程\停云小程序\tysj8
node .\start-dev-server.cjs
```

如果是在 Codex 里后台启动，使用上文的 `Invoke-CimMethod` 方式。  
这次的问题不是 Sparkle 虚拟网卡导致的，`127.0.0.1` 是本机回环地址，通常不走虚拟网卡。

## 后续升级建议

建议按这个顺序继续：

1. 完善各个集合的字段、下拉选项、图片预览。
2. 给订单、预约类集合做状态筛选和快捷改状态。
3. 把后台登录从当前账号密码升级为 CloudBase 用户名密码登录。
4. 用 CloudBase 用户 UID 做管理员白名单。
5. 最后再接微信扫码登录。

微信扫码登录需要额外配置微信开放平台或公众号网页授权、回调域名、CloudBase 登录方式，建议等后台功能稳定后再做。
