# 停云后台管理操作指南

## 1. 架构与安全

- 正式后台为 `tysj8` Vue 项目，业务管理通过 `tingyun2026/cloudfunctions/adminManage` 完成。
- 后台地址、账号、密码、CloudBase 环境 ID、管理员 token 均从部署平台或密码管理器获取，不记录在仓库文档。
- `ADMIN_USERNAME`、`ADMIN_API_TOKEN` 等仅配置为云函数环境变量；发现曾出现在文档、截图或提交记录中的凭据时应立即轮换。

## 2. 可管理模块

首页内容、介绍页、菜品、桌台二维码、用餐/住宿资源、活动、会员、会员权益、用餐/住宿预订、活动报名、点餐订单。

餐标菜品明细按分类维护，例如“凉菜 / 热菜 / 主食”。每个分类下录入具体菜名，可在后台对分类和单个菜名进行新增、上移、下移和删除；不要再把“热菜 8 道”这类描述写入菜品明细。

点餐订单重点字段：

- 点菜身份：`customer_type`、`customer_name`、`customer_mobile`。
- 结账身份：`checkout_customer_type`、`checkout_customer_name`、`checkout_customer_mobile`。
- 金额：整桌普通价、整桌会员价、实际结账金额。
- 状态：`preparing`/`completed` 与 `unpaid`/`paying`/`offline_pending`/`settled`。

## 3. 点餐操作规范

1. 新单和加菜已由小程序提交时通知厨房，不在后台重复打印。
2. 散客支付成功由回调自动将订单置为 `completed`、`settled` 并清台；后台无需再手动清台。
3. 会员在小程序确认积分抵扣后，订单显示 `completed` + `offline_pending`，桌台已自动释放；店员在线下核对完成后，在“点餐订单”列表点击“会员核销”，系统将支付状态改为 `settled`。
4. 桌台总览右侧“最近订单”支持“全部 / 会员待核销”筛选；会员待核销订单用于店员后续核销，不代表桌台仍占用。
5. 桌台总览的“编辑价格”可调整整桌总价和设置单个菜品免费；保存后会同步更新订单菜品、批次和整单金额。
6. 不要手工写入逐批支付、`pending_offline`、`settlement_status` 等旧字段。

## 4. 本地后台开发

```powershell
cd E:\AI编程\停云小程序\tysj8
npm install
npm run dev
```

后台集合白名单在 `cloudfunctions/adminManage/index.js`，页面字段在 `tysj8/src/config/modules.js`。新增可管理集合时需同时更新两处，并验证云函数权限。

## 5. 发布检查

- 变更 `adminManage` 或后台字段后，同时部署云函数和静态后台。
- 检查点餐双价格、结账身份、待核销/已结清筛选是否正确。
- 检查企业微信、订阅消息、厨房打印失败记录是否可在 `notification_logs` 中追踪。
