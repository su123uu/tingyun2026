# 通知配置与部署说明

本项目已接入三类通知：

- 用户侧：微信小程序订阅消息，用于点餐和预订状态变化提醒。
- 会员侧：微信小程序订阅消息（会员消费通知），用于会员消费记录提醒。
- 店员侧：企业微信应用消息，用于新订单、新预订和状态变更提醒。

## 1. 微信小程序订阅消息

### 后台操作

1. 打开微信小程序后台。
2. 进入「功能」或「订阅消息」。
3. 当前已使用三个模板：
   - 扫码点餐通知：用于扫码点餐、用餐预订状态。
   - 预定通知：用于住宿预订状态。
   - 会员消费通知：用于会员扫码点餐、预订住宿、预定用餐的消费提醒。
4. 记录每个模板里关键词对应的字段 key。

### 小程序端配置

编辑 `tingyun2026/config/notifications.js`：

```js
const templates = {
  mealOrderStatus: 'C4zcP7Aa--zKTf_hCrhfdBwtGt7g-NTN2V6yEQ0mJ3Q',
  reservationStatus: 'pxIcS6FOmd-u0Nw9p6n59FK1bqFBzEkYp39S7LmfRKk',
  memberConsumption: 't7Ae7NshuMt3CPQkEeati0WHdRG4jNuWc0WTa301rpM',

```


## 2. 企业微信店员提醒

### 企业微信后台操作

1. 企业微信管理后台创建「自建应用」。
2. 记录：
   - 企业 ID：`CorpID`
   - 应用 ID：`AgentID`
   - 应用 Secret
3. 设置应用可见范围，确保店员能收到消息。
4. 在开发者接口里配置云函数出口 IP 到可信 IP（如企业微信要求）。

### 云函数环境变量

给 `notificationManage` 云函数配置：

```text
WECOM_CORP_ID=企业ID
WECOM_AGENT_ID=应用AgentID
WECOM_APP_SECRET=应用Secret
WECOM_TO_USER=店员企业微信账号
```

多人可用竖线分隔：

```text
WECOM_TO_USER=zhangsan|lisi
```

也可以改用部门或标签：

```text
WECOM_TO_PARTY=部门ID
WECOM_TO_TAG=标签ID
```

## 3. 需要部署的云函数

至少部署：

- `notificationManage`
- `mealOrderManage`
- `reservationManage`
- `payCallback`
- `adminManage`
- `initDatabase`

首次上线前运行一次 `initDatabase`，它会创建：

- `notification_subscriptions`
- `notification_logs`

## 4. 验证流程

1. 填好小程序模板 ID。
2. 上传并部署云函数。
3. 重新编译小程序。
4. 散客提交点餐并完成微信支付后，应收到扫码点餐通知（状态：制作中）。
5. 会员提交点餐后，应同时收到扫码点餐通知（状态：制作中）和会员消费通知。
6. 会员提交预订后，应同时收到预订状态通知和会员消费通知。
7. 后台执行「完成并清台」把点餐改为「已完成」，用户应收到订阅消息（如果还有剩余配额）。
8. 后台把预订改为「已确认 / 未通过 / 已取消」，用户应收到订阅消息。
9. 创建新订单或预订时，店员应收到企业微信提醒。

如果用户没有点「允许」，或者模板字段 key 配错，订阅消息不会成功发送；失败原因会写入 `notification_logs`。
