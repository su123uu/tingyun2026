# 打开本地后台操作流程

本项目统一使用 `tysj8` 作为后台管理端。旧的 `admin-web` 静态后台已经删除，不再使用。

## 1. 进入后台目录

在 PowerShell 中进入项目后台目录：

```powershell
cd E:\AI编程\停云小程序\tysj8
```

## 2. 安装依赖

第一次运行，或 `node_modules` 不存在时执行：

```powershell
npm install
```

如果依赖已经安装过，可以跳过这一步。

## 3. 启动本地后台

推荐使用项目内置的稳定启动脚本：

```powershell
npm run dev:local
```

这个脚本会使用固定端口 `5173` 启动后台。

访问地址：

```text
http://localhost:5173/
```

## 4. 备用启动方式

也可以使用 Vite 默认开发命令：

```powershell
npm run dev
```

如果 `5173` 被占用，Vite 可能会自动切到 `5174`、`5175` 等端口。此时以终端输出的地址为准，例如：

```text
http://localhost:5174/
```

## 5. 在浏览器中打开

启动成功后，在浏览器地址栏打开：

```text
http://localhost:5173/
```

如果终端显示的是其它端口，例如 `5174`，就打开终端显示的地址。

## 6. 判断后台是否已经启动

可以在 PowerShell 中检查端口：

```powershell
netstat -ano | Select-String ':5173'
```

如果看到 `LISTENING`，说明本地后台正在运行。

也可以直接访问：

```powershell
Invoke-WebRequest -UseBasicParsing -Uri http://127.0.0.1:5173/ -TimeoutSec 5
```

返回状态码 `200` 表示页面可访问。

## 7. 关闭本地后台

如果是通过前台终端运行的 `npm run dev` 或 `npm run dev:local`，在该终端按：

```text
Ctrl + C
```

如果后台进程已经在后台运行，可以先查找端口对应进程：

```powershell
netstat -ano | Select-String ':5173'
```

找到最后一列的进程 ID 后停止它：

```powershell
Stop-Process -Id 进程ID -Force
```

示例：

```powershell
Stop-Process -Id 27272 -Force
```

## 8. 后续部署说明

当前阶段是本地调用和本地调试。等后台功能完成后，再执行 `tysj8` 的构建和部署流程，把后台发布到 CloudBase。

构建命令：

```powershell
npm run build
```
