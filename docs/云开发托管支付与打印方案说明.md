# 云开发支付与打印运行说明

## 架构

```text
小程序 / Vue 后台
  → 云函数
    → 云数据库
    → 微信支付统一下单与回调
    → 企业微信通知、订阅消息、芯烨云打印
```

静态托管只承载后台页面；支付密钥、企业微信密钥、打印机密钥、订单写入和回调处理必须在云函数环境执行。

## 点餐支付与打印

- 点餐提交时创建或追加整桌订单，立即打印本批次厨房单。
- 订单详情是唯一结账入口。散客按 `regular_total_amount` 发起一笔微信支付；会员确认积分抵扣后按 `member_total_amount` 写为 `completed + offline_pending`，同时清台，等待后台核销为 `settled`。
- `payCallback` 只结清整桌支付，不处理旧逐批支付。
- 微信支付回调必须按支付单号幂等：已结清订单不再次改变金额、打印或发送重复通知。

## 必需环境变量

```text
WECHAT_PAY_SUB_MCH_ID=
WX_CLOUD_ENV_ID=
WECHAT_PAY_CALLBACK_FUNCTION=payCallback

XPYUN_USER=
XPYUN_USER_KEY=
XPYUN_PRINTER_SN=
```

实际商户号、密钥和打印机编号仅在云函数环境变量配置，不提交仓库。

## 发布步骤

1. 在测试环境完成整桌支付回调、重复回调、取消支付和会员待核销测试。
2. 部署支付、通知、打印相关云函数。
3. 配置支付、企业微信、订阅模板和打印机环境变量。
4. 用真实桌码完成散客与会员两条链路验收。
