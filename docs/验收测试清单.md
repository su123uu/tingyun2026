# 停云山居验收测试清单

## 1. 身份与会员

- [ ] 未授权或未匹配手机号显示散客。
- [ ] 微信手机号授权后可匹配有效会员；会员中心不再使用固定测试手机号。
- [ ] 点餐页与确认订单页均以当前授权用户显示会员/散客身份，不被桌台会话覆盖。
- [ ] 小程序不展示余额、积分、积分流水或点餐自动扣减记录。
- [ ] 预订和活动联系人不预填固定测试手机号，只使用授权手机号或用户输入/保存联系人。

## 2. 扫码点餐与双价格

- [ ] 未扫码可浏览和加购，提交时提示扫码桌码。
- [ ] 首次扫码填写人数并创建桌台会话。
- [ ] 已开台且有活动订单的桌码再次扫码，直接打开该订单详情。
- [ ] 确认菜品页提交后不调起微信支付，首单/加菜写入同一个订单的 `batches`。
- [ ] 每次提交菜品只打印和通知本次新增批次。
- [ ] 订单详情展示每个菜品、每个批次及整桌的普通价、会员价。
- [ ] 未配置会员价时，会员价等于普通价。
- [ ] 散客结账按整桌普通价发起一次微信支付。
- [ ] 会员结账按整桌会员价弹出积分抵扣确认，确认后进入 `offline_pending`，不调微信支付。
- [ ] 会员开台/散客结账、散客开台/会员结账均按结账者身份与价格处理。
- [ ] 散客微信支付成功后订单自动完成并清台。
- [ ] 会员确认积分抵扣后订单为 `completed + offline_pending`，桌台立即清空；后台完成线下核销后支付状态变为 `settled`。
- [ ] 支付取消仅恢复整桌为 `unpaid`；重复支付回调不重复结账、通知或打印。

## 3. 消息、打印与后台

- [ ] 点餐提交：客户收到制作中订阅消息（已授权时），企业微信收到新单/加菜，厨房收到本批次打印。
- [ ] 散客支付成功：企业微信收到已结账；客户不会因同一份已使用授权重复收到消息。
- [ ] 会员结账：企业微信收到待核销；后台核销完成后企业微信收到核销完成，客户收到会员消费消息（已授权时）。
- [ ] `notification_logs` 可追踪微信订阅消息与企业微信发送成功/失败。
- [ ] 后台订单列表展示点菜身份、结账身份、普通价、会员价、实际结账金额和统一支付状态。
- [ ] 后台桌台总览“最近订单”可筛选会员待核销订单；会员核销完成后该订单从筛选中消失。
- [ ] 后台桌台总览展示菜品总价，可编辑整桌总价和单菜免费状态。
- [ ] 后台完成会员核销不会写入旧点餐状态或逐批支付字段。

## 4. 预订与活动回归

- [ ] 用餐预订人数、包间容量、餐标金额、房态冲突校验正确。
- [ ] 住宿散客按普通房价、会员按会员房价；免费住宿权益按锁定/核销/释放处理。
- [ ] 预订和活动会员待核销状态为 `offline_pending`。
- [ ] 收费散客支付回调、活动名额、免费活动取消、后台状态通知正常。

## 5. 发布前检查

- [ ] 部署 `mealOrderManage`、`payCallback`、`notificationManage`、`adminManage`、`reservationManage`、`activitySignupManage`、`initDatabase`。
- [ ] 重新构建小程序 npm，确认 TDesign 组件可加载。
- [ ] 云函数环境变量、微信模板字段、企业微信应用/Webhook、打印机配置均已验证。
