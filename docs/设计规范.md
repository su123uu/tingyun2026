# 停云山居小程序 · 设计规范 v2.4

> 更新日期：2026-06-01
> 适用范围：HTML 视觉原型 + 后续微信小程序落地
> 文档边界：只记录视觉、布局和组件表现；页面功能、业务规则和跳转逻辑统一写入需求文档

---

## 一、设计定位

| 维度 | 规范 |
|------|------|
| 风格关键词 | 典雅、安静、留白、东方、克制 |
| 视觉气质 | 山间生活空间，不做强促销和连锁茶饮感 |
| 页面密度 | 首屏保留关键视觉信号，模块之间有呼吸感 |
| 参考方式 | 借鉴成熟小程序的信息效率，不直接复刻具体品牌 |
| 实现方式 | HTML 原型先行，确认后迁移至小程序 WXML/WXSS |

---

## 二、色彩系统

| 名称 | 色值 | 用途 |
|------|------|------|
| 山居绿 | `#4F6F52` | 主按钮、导航选中态、品牌强调 |
| 古铜棕 | `#8B7355` | 预约相关模块、辅助强调 |
| 宣纸白 | `#F8F6F1` | 页面底色 |
| 暖米色 | `#E8E4DC` | 卡片底、图片占位、浅色分隔 |
| 装饰金 | `#C9A962` | 细线、徽章、小面积会员强调 |
| 墨黑 | `#1A1A1A` | 主标题、价格、关键数字 |
| 墨灰 | `#4A4A4A` | 正文、说明文字 |
| 雾灰 | `#7C8A8C` | 辅助说明、空状态、次级图标 |
| 禁用灰 | `#BBBBBB` | 不可点击、未完成状态 |

色彩原则：

- 金色只做细线、徽章和少量强调，不做大面积背景。
- 页面背景保持浅暖，不使用冷蓝、亮紫和强商业渐变。
- 单个页面的主色控制在 1-2 个，避免视觉噪音。
- 深色卡片可以少量使用，但需要克制，避免变成促销广告。

## 三、字体与排版

HTML 原型可使用系统字体或 Noto Serif SC 作为品牌标题参考；微信小程序落地以系统字体为主，通过字重、间距和留白体现东方感。

| 用途 | HTML 建议 | 小程序建议 | 说明 |
|------|-----------|------------|------|
| 页面标题 | 22-28px | 40-48rpx | 首页主视觉、详情页标题 |
| 区块标题 | 16-18px | 30-34rpx | 页面分区标题 |
| 卡片标题 | 15-17px | 28-32rpx | 商品名、活动名、房间名 |
| 正文 | 13-15px | 24-28rpx | 内容说明 |
| 辅助文字 | 11-13px | 20-24rpx | 时间、状态、提示、标签 |
| 价格/数字 | 18-24px | 30-42rpx | 金额、统计数据 |

排版原则：

- 常规页面左右安全边距建议 14-16px / 28-32rpx。
- 模块间距建议 16-24px / 32-48rpx。
- 卡片内部标题和正文间距控制在 6-10px / 12-20rpx。
- 不使用 emoji 作为正式图标，HTML 原型也尽量使用 SVG 线性图标。
- 文字与容器保持足够留白，不依赖过大的圆角制造视觉层次。

## 四、圆角与阴影

整体圆角保持适中，避免过度柔软。

| 组件 | HTML 建议 | 说明 |
|------|-----------|------|
| 首页内容卡 | 14px | 山居介绍、会员活动 |
| 首页快捷入口卡 | 18px | 点餐/预约组合卡 |
| 常规内容卡 | 12-16px | 商品、订单、空间、权益 |
| 小标签 | 10-14px | 状态、标签 |
| 圆形按钮 | 50% | 加减数量、图标按钮 |

统一使用轻阴影：

```css
box-shadow: 0 8px 24px rgba(26, 26, 26, 0.045);
```

原则：

- 阴影只用于区分层级，不做厚重悬浮感。
- 同一页面避免多种强弱差异过大的阴影。
- 卡片中不要再嵌套视觉上完整的卡片。

## 五、首页视觉规范

首页作为整体视觉基准，结构固定为：

| 顺序 | 模块 | 视觉规范 |
|------|------|----------|
| 1 | 顶部轮播 | 3 张以内，宽高比 `4:5`，顶到页面上方，无外边距、无圆角 |
| 2 | 快捷入口 | 两个同尺寸入口，放在同一张白色功能卡中 |
| 3 | 山居介绍 | 一张图片宽高比 `2.43:1` 1388 × 572px+ 一个主标题 |
| 4 | 会员活动 | 一张图片宽高比 `2.43:1` 1388 × 572px+ 一个主标题 |

尺寸原则：

- 快捷入口功能卡左右边距约 14px，上压轮播约 50px。
- 两个快捷入口左右等宽，中间使用细分隔线。
- 山居介绍和会员活动卡左右边距与快捷入口卡一致。
- 山居介绍和会员活动卡片圆角约 14px。
- 图片和主标题之间使用明确的白色标题栏。
- 首页内容卡不放副标题、正文和多余标签。

## 六、底部导航视觉规范

所有主页面统一使用 5 个底部 Tab：

`首页 | 点餐 | 介绍 | 住宿 | 我的`

视觉规则：

- 背景使用半透明白或宣纸白，顶部加细分隔线。
- 选中态统一使用山居绿，不用金色作为主选中态。
- 图标使用统一粗细的线性图标。
- 图标尺寸约 22-25px，文字约 11-12px。
- 底部导航高度需要包含安全区，不遮挡页面滚动内容。

## 七、业务页面视觉模板

### 点餐页

- 左侧分类栏 + 右侧商品列表。
- 商品卡片包含商品图、名称、简短描述、价格和数量控制。
- 底部购物车栏固定悬浮，显示数量、合计和主操作按钮。
- 未激活状态使用禁用灰，不隐藏主要入口。

### 确认订单页

- 使用单列信息结构，依次展示桌号、商品明细、金额、备注和底部主按钮。
- 商品明细延续点餐页的图片和数量表达。
- 总金额需要有明显但克制的强调。

### 订单页

- 订单卡片使用“订单头部 + 商品图/商品摘要 + 订单信息 + 操作按钮”结构。
- 商品图建议 64-76px 方形，圆角 10-12px。
- 状态标签放在卡片右上角，颜色统一遵循状态色。

### 介绍页

- 以图片和留白为主，避免堆叠过多文字。
- 空间、住宿、餐厅信息保持统一卡片样式。
- 联系方式放在页面末尾，视觉权重低于内容展示。

### 预约大厅

- 住宿、用餐、空间、举办活动使用同一类入口卡片。
- 卡片包含氛围区、标题、短说明、少量标签和操作提示。
- 不同入口可使用同一色系中的轻微差异，避免四色拼盘感。

### 我的页

- 会员信息使用清晰的信息层级：用户信息、等级、折扣、累计消费、权益。
- 金色仅用于等级和权益强调。
- 功能入口使用简洁列表，不做密集宫格。

### 活动页

- 头图 + 活动标题 + 内容卡片。
- 活动列表保持图片、时间、主标题和简短说明的统一结构。
- 主要行动按钮放在页面末尾或底部固定区。

---

## 八、状态视觉规范

| 状态 | 颜色 |
|------|------|
| 待支付 / 待处理 | 装饰金 |
| 已支付 / 制作中 | 山居绿 |
| 已完成 | 雾灰 |
| 已取消 | 禁用灰 |

状态表达原则：

- 状态标签使用浅底色 + 深色文字。
- 禁用按钮使用禁用灰或低透明度，但仍应可识别。
- 空状态使用简洁线性图标和一句说明，不使用夸张插画。

---

## 九、原型迁移规范

HTML 原型是当前视觉设计稿，迁移到微信小程序时遵循：

- 保持页面结构、间距比例、字体层级和组件样式一致。
- HTML 中的 SVG 线性图标迁移为 TDesign 图标或等价自定义图标。
- HTML 中的 `px` 按视觉比例换算为小程序 `rpx`。
- HTML 原型中的弹窗提示、假数据和模拟状态不视为后端功能完成。
- 业务规则、页面跳转和数据字段以需求文档为准。

## 十、微信小程序端适配规范

### 自定义导航栏
- 需要沉浸式顶部视觉或自定义标题布局的页面使用 `navigationStyle: "custom"`。
- 首页轮播图直接延伸到页面顶部，不显示系统白色导航栏。
- 点餐页保留白色标题区域，但由页面自定义绘制，不使用系统导航栏标题。
- 点餐页标题“山居点餐”与微信右上角胶囊按钮处于同一水平行，副标题放在标题下一行。
- 标题区必须通过 `wx.getMenuButtonBoundingClientRect()` 动态读取胶囊按钮的 `top` 和 `height`，不得仅依赖固定像素或单一机型截图。
- 自定义标题区右侧应预留胶囊按钮空间，避免标题、按钮和胶囊发生重叠。
- 无法读取胶囊尺寸时，使用 `statusBarHeight` 计算降级位置。

### 点餐页固定结构
- 点餐页根容器高度固定为 `100vh`，页面本身不整体滚动。
- 顶部标题区固定，桌号卡片紧接标题区固定展示。
- 左侧分类栏固定，不随商品列表滚动。
- 右侧商品列表为独立滚动容器，滚动时标题、桌号和分类栏保持可见。
- 购物车栏固定悬浮在底部导航上方，位置需要叠加 `env(safe-area-inset-bottom)`，避免被全面屏安全区和自定义 TabBar 遮挡。
- 点击购物车非结算区域展开已选菜品面板；面板内支持继续增减数量；点击遮罩或“收起”关闭面板。
- “去结算”按钮保持独立点击行为，不触发购物车展开。

### 底部安全区
- 自定义 TabBar、固定结算栏、购物车栏和底部弹层都必须包含 `env(safe-area-inset-bottom)`。
- 固定悬浮组件与 TabBar 之间保留明确间距，不得依靠单一机型的固定 `bottom` 数值判断位置。
