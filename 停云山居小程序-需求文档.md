## 一、项目概述

### 1.1 项目背景

停云山居是一家位于青岛崂山的文化空间，集禅修、茶艺、美食、住宿于一体。本次开发微信小程序，旨在为散客提供便捷的扫码点单和付费体验，同时展示山居文化、会员权益和活动信息。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 散客扫码点单 | 客人入座后扫桌上二维码 → 浏览菜单 → 加入购物车 → 微信支付 → 后厨接单 |
| 山居文化展示 | 通过精美页面展示停云山居的环境、理念、特色服务 |
| 会员体系 | 微信手机号一键注册成为会员，查看权益和等级信息 |
| 活动管理 | 展示当前/往期活动，引导用户通过电话或客服咨询报名 |

### 1.3 设计说明

页面视觉、配色、字体、间距、圆角和组件样式统一以《设计规范.md》为准。本文件只记录产品需求和业务规则。

---

## 二、用户角色

| 角色 | 说明 | 核心需求 |
|------|------|---------|
| **散客（游客）** | 到店用餐的客人，无需注册即可点单 | 扫码→点菜→支付→查看订单 |
| **注册会员** | 已用微信手机号注册的客人 | 查看会员权益和等级、报名活动、查看历史订单 |

---

## 三、全局导航

所有主页面底部统一使用 5 个 Tab：

`首页 | 点餐 | 介绍 | 住宿 | 我的`

其中“住宿”Tab 打开预约大厅，预约大厅再展示住宿、用餐、空间预定和举办活动四类入口。

### 模块 A：首页（Home）

**首页定稿内容**：

| 区域 | 功能 | 优先级 |

**跳转关系**：
- 扫码点单 → 点单页
- 预约预订 → 预约展示页
- 山居介绍卡 → 介绍页
- 会员活动卡 → 活动页

**当前状态**：HTML 原型首页作为视觉基准，后续迁移到小程序页面。

### 模块 B：扫码点单（Menu）

**页面位置**：小程序底部Tab第2项 / 扫桌上二维码直达

**扫码点单完整流程**：

```
客人扫桌上二维码
        ↓
解析二维码中的桌号（如 A01）
        ↓
进入点单页面，顶部显示"桌号 A01"
        ↓
浏览菜品 → 加入购物车 → 去结算 → 确认订单 → 微信支付 → 下单成功
```

**结算规则**：
- 只有扫描桌上二维码并成功解析桌号后，才能结算。
- 从首页或底部 Tab 直接进入时，允许浏览菜单和加购，但结算按钮不可用。
- 用户扫码进入点餐页后填写用餐人数；该数据同步到确认订单页和订单详情页。
- 用户点击结算后进入独立的确认订单页，不使用底部弹层。
- HTML 阶段使用模拟支付成功流程，真实支付后续接入。

**菜品分类**：

| 分类 | 包含菜品 |
|------|---------|
| 套餐 | 下午茶套餐(38元)、林鸡野趣套餐(168元)、泉鱼隐逸套餐(198元)、双鲜套餐(318元) |
| 凉菜 | 黄瓜拌(18元) |
| 热菜 | 干煸崂山参(48元) |
| 主食 | 海鲜水饺(38元) |
| 饮品 | 桂花酒(28元) |

**购物车功能**：添加/减少数量、展开详情、实时合计、去结算

**目标支付流程**：确认订单 → 调起微信支付 → 支付成功跳转详情页

**当前状态**：已有 HTML 基础原型，待按最新需求重做视觉与结算流程。

### 模块 C：确认订单（Order Confirm）

**入口**：点餐页购物车 → 去结算

**进入条件**：必须已通过桌上二维码获得桌号，且购物车内至少有 1 件商品。

**页面内容**：

| 区域 | 内容 |
|------|------|
| 就餐信息 | 桌号、堂食标识、用餐人数 |
| 商品明细 | 商品图片、名称、单价、数量、小计 |
| 订单备注 | 自由输入框 + 快捷标签 |
| 金额汇总 | 商品金额、优惠金额、应付金额 |
| 底部操作栏 | 应付金额、微信支付按钮 |

**备注规则**：
- 支持自由输入口味、忌口和其他需求。
- 提供快捷标签，例如：少辣、不要香菜、有儿童、稍后上菜。
- 用户可以组合选择快捷标签，并继续补充自由输入内容。

**HTML 模拟支付流程**：

```
点击“微信支付”
        ↓
显示“支付成功”
        ↓
显示“订单已提交，正在通知厨房”
        ↓
进入订单详情页
```

**当前阶段不做**：真实微信支付、真实会员折扣、优惠券、发票、库存校验、后厨打印状态同步。

### 模块 D：订单管理（Orders）

**入口**：点餐支付成功后 / 个人中心“点餐订单” / 个人中心“预约订单”

**订单分类**：

| 类型 | 内容 |
|------|------|
| 点餐订单 | 展示桌上扫码点餐记录 |
| 预约订单 | 展示餐厅预订、住宿预订；未来接入空间预定、举办活动 |

**通用功能**：订单列表（按时间倒序）、状态筛选、订单摘要、订单详情、联系客服、符合条件时取消订单。

### 模块 E：订单详情（Order Detail）

**入口**：
- 点餐支付成功后自动进入订单详情页。
- 订单列表点击“查看详情”进入订单详情页。
- 预约提交成功后进入对应订单详情页。

**实现方式**：点餐订单、餐厅预定、住宿预定共用一个订单详情页模板，根据订单类型展示不同字段。未来空间预定和举办活动也复用此模板。

**公共区域**：

| 区域 | 内容 |
|------|------|
| 状态区 | 当前状态、状态说明 |
| 基础信息 | 订单编号、创建时间、订单类型 |
| 金额信息 | 支付方式、已付金额或到店支付提示 |
| 备注信息 | 用户备注 |
| 操作区 | 联系客服、符合条件时显示取消订单 |

**点餐订单专属字段**：

| 区域 | 内容 |
|------|------|
| 就餐信息 | 桌号、用餐人数 |
| 商品明细 | 商品图片、名称、数量、小计 |
| 支付信息 | 商品金额、微信支付金额、会员账户核对状态、支付时间 |
| 厨房状态 | 待支付 → 支付成功，正在通知厨房 → 已通知厨房 → 制作中 → 已完成 |

**点餐订单规则**：
- 支付成功后自动触发云打印和接单通知。
- 云打印成功后自动进入“已通知厨房”。
- 厨房或店员只需操作“制作中”和“已完成”。
- 打印或通知异常仅在后台提示，用户端不展示技术错误。

**餐厅预定专属字段**：

| 区域 | 内容 |
|------|------|
| 餐厅信息 | 日期、午餐或晚餐时段、人数、已选包间（可多选） |
| 联系信息 | 联系人、手机号 |
| 支付提示 | 散客微信支付；会员由客服线下核对会员账户 |
| 状态 | 待店长确认 → 已确认 / 未通过 → 已完成；用户也可取消 |

**住宿预定专属字段**：

| 区域 | 内容 |
|------|------|
| 房间信息 | 已选具体房间（可多选）、房型 |
| 入住信息 | 入住日期、退房日期、住宿晚数、入住人数 |
| 联系信息 | 联系人、手机号 |
| 支付提示 | 散客微信支付；会员由客服线下核对会员账户或核销次数权益 |
| 状态 | 待店长确认 → 已确认 / 未通过 → 已完成；用户也可取消 |

**预约确认规则**：
- 餐厅预定和住宿预定提交后，不直接视为成功，先显示“待店长确认”。
- 客服需要电话联系用户，确认具体需求和订单信息。
- 空间预定和举办活动未来接入后，也沿用“提交需求 → 客服联系 → 店长确认”的流程。
- 店长拒绝预约时，订单状态为“未通过”，并填写原因。常见原因包括包间已占用、房间冲突、人数不符合要求、日期不可用。

**取消规则**：
- 点餐订单支付前允许取消。
- 点餐订单支付成功后不允许用户自行取消，只能联系客服处理。
- 餐厅预定和住宿预定在“待店长确认”阶段允许用户取消。
- 预约订单确认后不允许用户自行取消，只能联系客服处理。

### 模块 G：会员中心（Member）

**入口**：个人中心 → 个人信息卡片下方的黑金"成为会员"入口

#### G.1 办理方式
- 会员卡只能在线下联系店长购买。
- 小程序不提供在线购卡、续卡、升档、退款、转赠或提现入口。
- 会员介绍页展示会员权益，并提供“联系店长办理”入口。

#### G.2 五级会员卡

| 等级 | 名称 | 线下办卡金额 | 线下账户额度 | 权益有效期 |
|------|------|---------:|---------:|-----------:|
| 1 级 | 将星 | 1 万元 | 1.2 万元 | 12 个月 |
| 2 级 | 辰升 | 2 万元 | 2.5 万元 | 16 个月 |
| 3 级 | 海尊 | 3 万元 | 3.7 万元 | 18 个月 |
| 4 级 | 山王 | 5 万元 | 6.2 万元 | 24 个月 |
| 5 级 | 云境 | 10 万元 | 12.5 万元 | 36 个月 |

#### G.3 已入会会员中心字段

**会员卡概览**：
- 用户头像、昵称、手机号。
- 当前等级：将星 / 辰升 / 海尊 / 山王 / 云境。
- 会员卡状态：生效中、即将到期、已到期。
- 权益生效日期、到期日期。
- 小程序不展示积分余额或线下账户余额。
- 联系店长入口：用于办理、续卡、升档。

**线下账户规则**：
- 会员余额、积分和消费扣减由线下办卡软件管理。
- 小程序不展示积分余额、线下账户余额、积分明细或积分抵扣记录。
- 点餐、预约和活动订单中，会员只提交需求；客服/店员在线下系统核对会员账户。
- 小程序只展示会员身份、服务权益和次数权益；次数权益可由后台维护和核销。

**会员账户记录**：
- 小程序不记录线下账户流水。
- 后台仅保留订单的会员账户核对状态、权益核销状态和必要备注。
- 真实余额、积分和扣减记录以线下办卡软件为准。

#### G.4 十大会员权益

所有等级均享有十大权益。权益在会员卡有效期内生效，到期后未使用额度清零。续卡后按新周期重新发放，不累计旧周期额度。

| 序号 | 权益 |
|------|------|
| 1 | 山居资源圈与跨界活动 |
| 2 | 免费住宿 10 次 |
| 3 | 月度会员专题活动 |
| 4 | 禅修营 5 人次 / 年、子女自然教育课程 5 人次 / 年 |
| 5 | 每年 4 次禅修营专题音乐会 |
| 6 | 山居管家服务 |
| 7 | 私人纪念日主题布置 3 次 |
| 8 | 茶室 10 次、茶酒吧 10 次、多功能会议室或禅修室 1 次 |
| 9 | 存茶、存雪茄、存酒等定制储藏服务 |
| 10 | 果园农场自然采摘 20 人次 / 年 |

会员中心逐项展示：
- 权益名称。
- 总次数或额度。
- 已使用次数、剩余次数。
- 到期日期。
- 核销记录。

#### G.5 我的会员活动
- 活动参与资格：仅权益有效期内的会员可以报名，会员卡到期后不可报名。
- 展示已报名活动和历史参与记录。
- 展示活动标题、时间、地点、参与人数。
- 状态：待确认、已确认、已完成、已取消。

### 模块 H：活动管理（Activities）

**入口**：首页会员活动卡片 → 点击进入活动页

**功能**：

| 功能 | 优先级 | 当前状态 |
|------|--------|---------|
| 活动列表展示 | P0 | ✅ 已有基础 |
| 活动图文详情 | P0 | ✅ 已有基础 |
| 用户直接报名 | P0 | 待制作 |
| 电话 / 微信客服 / 微信群报名 | P0 | 由店员后台补录 |

**活动列表字段**：封面图、活动标题、日期、地点、剩余名额、报名状态（可报名、已满员、已截止）。

**活动详情字段**：图文介绍、活动时间、地点、人数上限、剩余名额、报名截止时间、活动须知、报名按钮、联系客服入口。

**报名表字段**：姓名、手机号、参与人数、备注。姓名和手机号默认从会员信息带出，手机号允许修改。

**报名规则**：
- 仅权益有效期内的会员可以报名。
- 用户提交后状态为“待确认”，店员确认后改为“已确认”。
- 每位会员默认最多报名 2 人：会员本人 + 1 位同行者。超过人数需要联系店员沟通。
- 用户可以自行取消，取消后释放活动名额并保留“已取消”记录。
- 每场活动支持后台设置人数上限和报名截止时间。
- 不设置候补名单。
- 用户也可以通过电话、微信客服或微信群报名，由店员后台录入统一的活动名单。
- 采摘活动按实际参与人数扣减“自然采摘 20 人次 / 年”权益。

**后台活动字段**：活动标题、封面图、详情图、活动时间、地点、活动内容、人数上限、报名截止时间、报名状态、联系方式。

### 模块 J：预约大厅（Booking）

**入口**：首页“预约预订”入口 / 底部“住宿”Tab

**包含内容**：

| 类型 | 说明 | 当前阶段 |
|------|------|----------|
| 餐厅预订 | 日期、午餐或晚餐、人数、包间多选、联系人、备注 | v1 优先完善 |
| 住宿预订 | 入住离店日期、人数、具体房间多选、联系人、备注 | v1 优先完善 |
| 空间预定 | 展示会议室、禅修室、VIP 茶室、展厅入口 | 暂缓细化 |
| 举办活动 | 展示全场包场、茶会沙龙、企业团建、课程活动入口 | 暂缓细化 |

**支付规则**：预约相关均为到店支付，不做在线支付。

#### J.1 餐厅预订规则
- 用户填写用餐人数。
- 可选餐段：午餐 `11:00-14:00`、晚餐 `17:00-21:00`。
- 用户自行选择具体包间，支持多选。
- 每个包间最低预订人数为 6 人；不足 6 人时提示联系客服沟通。
- 当前 4 个包间的建议容纳人数分别为：`6-8`、`10-12`、`10-12`、`10-12` 人。
- 同一包间、同一日期、同一餐段只能存在一笔已确认预订。
- 不提供提前点菜功能。
- 当前不设置线上定金和在线支付。
- 到店结算时由客服/店员在线下办卡软件核对会员账户。
- 4 个包间的正式名称待补充。

#### J.2 住宿预订规则
- 共 7 个房间，名称、房间 ID 和人数上限以《房间ID与名称对照表.md》为准。
- 用户填写入住人数。
- 用户直接选择具体房间，支持多选。
- 填写入住日期、离店日期、联系人、手机号和备注。
- 最少入住 1 晚。
- 允许儿童入住。
- 不提供加床。
- 所选房间在入住日期范围内不可与其他已确认订单冲突。
- 散客微信支付；会员由客服/店员线下核对会员账户。

## 七、用户端原型制作清单

### 7.1 基础页面

| 页面 | 重点内容 |
|------|----------|
| 首页 | 轮播、点餐入口、预约入口、山居介绍、会员活动 |
| 我的 / 个人中心 | 头像昵称、成为会员、点餐订单、预约订单、活动权益、优惠券占位、客服、关于停云、设置 |
| 会员介绍 | 五级会员卡、十大权益、联系店长办理 |
| 已入会会员中心 | 会员卡、服务权益、权益次数、核销记录、我的会员活动 |
| 山居介绍 | 山居概况、空间、住宿、餐厅、活动环境、联系方式、地图导航 |

### 7.2 点餐流程

| 页面 | 重点内容 |
|------|----------|
| 点餐页 | 商品分类、商品、购物车；未扫码时可以浏览，只有桌上扫码后才能结算 |
| 确认订单 | 桌号、人数、商品、备注快捷标签、散客微信支付、会员账户核对提示 |
| 点餐订单列表 | 商品图、金额、时间、状态 |
| 点餐订单详情 | 商品明细、微信支付金额、会员账户核对状态、厨房状态 |

点餐状态：`待支付 → 支付成功，正在通知厨房 → 已通知厨房 → 制作中 → 已完成`

### 7.3 预约流程

| 页面 | 重点内容 |
|------|----------|
| 预约大厅 | 餐厅预订、住宿预订、空间预定、举办活动 |
| 餐厅预订 | 日期、午餐或晚餐、人数、包间多选、联系人、备注 |
| 住宿预订 | 入住离店日期、人数、具体房间多选、联系人、备注 |
| 预约订单列表 | 区分餐厅和住宿，展示日期与状态 |
| 预约订单详情 | 联系信息、预约内容、散客微信支付、会员账户核对/权益核销说明、取消规则 |

预约状态：`待店长确认 → 已确认 / 未通过 → 已完成`。用户也可在待确认阶段自行取消。

### 7.4 会员活动流程

| 页面 | 重点内容 |
|------|----------|
| 活动列表 | 封面、标题、日期、地点、剩余名额、状态 |
| 活动详情 | 图文、活动须知、截止时间、报名按钮、客服入口 |
| 活动报名 | 姓名、手机号、参与人数、备注 |
| 我的会员活动 | 待确认、已确认、已完成、已取消、历史记录 |
