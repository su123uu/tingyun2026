# 停云山居小程序接口设计 v1.0

> 制定日期：2026-06-01
> 当前阶段：前端 mock 服务。
> 未来阶段：保持页面调用参数不变，将 `tingyun2026/services/` 内部实现替换为云函数调用。

## 一、统一约定

- 页面只能调用 `services/`，不要直接读取 `mock/` 或操作本地存储。
- 服务方法统一返回 `Promise`。
- 成功时直接返回业务数据。
- 失败时抛出 `Error`，错误对象包含稳定的 `code`。
- 当前微信支付仅为前端模拟，不能视为真实支付凭证。
- 用户端展示的业务编号统一使用 `TYMEAL`、`TYDINING`、`TYROOM`、`TYACTIVITY` 前缀。

## 五、桌台与购物车

### 5.1 桌台配置与二维码

后台或管理端能力：

| 方法 | 参数 | 返回 | 用途 |
|------|------|------|------|
| `listDiningTables()` | `{ include_disabled }` | 桌台数组 | 查看桌台列表 |
| `createDiningTable(input)` | `{ table_id, table_name, table_area, capacity }` | 桌台 | 新增桌台 |
| `updateDiningTable(input)` | `{ table_id, table_name, table_area, capacity, table_status }` | 桌台 | 修改桌台信息或启停 |
| `deleteDiningTable(input)` | `{ table_id }` | 桌台 | 软删除桌台 |
| `generateDiningTableQr(input)` | `{ table_id }` | `{ table, qr_image_file_id, qr_scene }` | 生成或重新生成可打印二维码 |

未来云函数建议：`diningTableList`、`diningTableCreate`、`diningTableUpdate`、`diningTableDelete`、`diningTableQrGenerate`。

### 5.2 桌台会话 `services/table-session.js`

| 方法 | 参数 | 返回 | 用途 |
|------|------|------|------|
| `parseTableCode(code)` | `{ code }` | `{ table_id, qr_token }` | 解析桌上二维码 |
| `startTableSession(input)` | `{ code, people_count }` | 桌台会话 | 扫码后建立用餐会话 |
| `getCurrentTableSession()` | 无 | 桌台会话或 `null` | 获取当前桌台 |

前端 mock 可继续兼容旧二维码格式：`TY_TABLE:<桌号>`，例如 `TY_TABLE:A01`。

正式云端二维码建议格式：`TY_TABLE:<桌号>:<qr_token>`；云函数必须校验 `meal_tables.table_status = enabled` 且 `qr_token` 匹配，才允许创建桌台会话。

## 十、稳定错误码

| 错误码 | 含义 |
|------|------|
| `INVALID_MOBILE` | 手机号格式错误 |
| `INVALID_TABLE_CODE` | 桌码格式错误 |
| `TABLE_SESSION_REQUIRED` | 未扫码建立桌台会话 |
| `EMPTY_CART` | 购物车为空 |
| `ITEM_NOT_AVAILABLE` | 菜品未上架或暂不可售 |
| `INVALID_DATE_RANGE` | 日期范围错误 |
| `DINING_MIN_PEOPLE` | 用餐预订少于 6 人 |
| `ROOM_NOT_AVAILABLE` | 房间已占用或不可用 |
| `ROOM_CAPACITY_NOT_ENOUGH` | 所选房间总容量不足 |
| `ROOM_SELECTION_LIMIT` | 所选包间数量不符合规则 |
| `MEMBERS_ONLY` | 活动仅限会员 |
| `ACTIVITY_FULL` | 活动名额不足 |
| `ACTIVITY_SIGNUP_LIMIT` | 单次报名超过 2 人 |
| `SELF_CANCEL_NOT_ALLOWED` | 该订单需联系客服取消 |

## 十一、云服务接入时增加的能力

- 微信手机号解密和会员档案真实匹配。
- 云数据库权限、索引和并发事务。
- 住宿与用餐包间实时占用。
- 散客支付前不占用房间；支付成功时最终校验并占用。
- 微信支付、退款回调和交易核验。
- 财务流水生成、查询和导出。
- 厨房通知、云打印和异常重试。
- 后台人工确认、拒绝、取消、会员账户核对和权益核销登记。
