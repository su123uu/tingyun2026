# 停云山居小程序接口设计 v1.0

> 制定日期：2026-06-01
> 当前阶段：前端 mock 服务。
> 未来阶段：保持页面调用参数不变，将 `tingyun2026/services/` 内部实现替换为云函数调用。

## 一、统一约定

- 页面只能调用 `services/`，不要直接读取 `mock/` 或操作本地存储。
- 服务方法统一返回 `Promise`。
- 成功时直接返回业务数据。
- 失败时抛出 `Error`，错误对象包含稳定的 `code`。
- 当前微信支付仅为前端模拟，不能视为真实支付凭证。

## 二、用户身份 `services/auth.js`

| 方法 | 参数 | 返回 | 用途 |
|------|------|------|------|
| `getCurrentUser()` | 无 | 用户对象 | 获取当前身份 |
| `bindMobile(mobile)` | `{ mobile }` | 用户对象 | 模拟微信手机号授权并匹配会员 |
| `useGuestMode()` | 无 | 用户对象 | 清除绑定，切回散客 |

未来云函数建议：`authBindMobile`、`authGetProfile`。

## 四、桌台与购物车

### 4.1 桌台 `services/table-session.js`

| 方法 | 参数 | 返回 | 用途 |
|------|------|------|------|
| `parseTableCode(code)` | `{ code }` | `{ table_id }` | 解析桌上二维码 |
| `startTableSession(input)` | `{ code, people_count }` | 桌台会话 | 扫码后建立用餐会话 |

前端 mock 二维码格式：`TY_TABLE:<桌号>`，例如 `TY_TABLE:A01`。

### 4.2 购物车 `services/cart.js`

| 方法 | 参数 | 返回 | 用途 |
|------|------|------|------|

## 八、稳定错误码

| 错误码 | 含义 |
|------|------|
| `INVALID_MOBILE` | 手机号格式错误 |
| `INVALID_TABLE_CODE` | 桌码格式错误 |
| `TABLE_SESSION_REQUIRED` | 未扫码建立桌台会话 |
| `EMPTY_CART` | 购物车为空 |
| `ITEM_NOT_AVAILABLE` | 菜品未上架或暂不可售 |
| `INVALID_DATE_RANGE` | 日期范围错误 |
| `DINING_MIN_PEOPLE` | 用餐预订少于 6 人 |
| `ROOM_NOT_AVAILABLE` | 房间已占用或不可用 |
| `ROOM_CAPACITY_NOT_ENOUGH` | 所选房间总容量不足 |
| `ROOM_SELECTION_LIMIT` | 所选包间数量不符合规则 |
| `MEMBERS_ONLY` | 活动仅限会员 |
| `ACTIVITY_FULL` | 活动名额不足 |
| `ACTIVITY_SIGNUP_LIMIT` | 单次报名超过 2 人 |
| `SELF_CANCEL_NOT_ALLOWED` | 该订单需联系客服取消 |

## 九、云服务接入时增加的能力

- 微信手机号解密和会员档案真实匹配。
- 云数据库权限、索引和并发事务。
- 住宿与用餐包间实时占用。
- 散客支付前 15 分钟锁定及自动释放。
- 微信支付、退款回调和交易核验。
- 厨房通知、云打印和异常重试。
- 后台人工确认、拒绝、取消和积分扣款登记。
