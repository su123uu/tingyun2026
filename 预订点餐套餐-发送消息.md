# 预订/点餐店员转发文案

> 更新日期：2026-06-24  
> 用途：供店员在企业微信中手工复制发送给客户；不是系统自动通知模板。自动通知配置见《通知配置与部署说明》。

## 使用规则

- 必须以后台订单详情为准，替换订单号、联系人、日期、金额和状态后再发送。
- 点餐确认仅表示订单已提交；整单结账后才可写“已结账”。
- 会员结账显示“待线下核销”时，不写“已扣款”或“已结账”。
- 不在企业微信中发送客户完整身份证件、支付凭证或其他敏感信息。

## 预定确认

```text
【停云山居】
{联系人}，您好！
已为您登记{套餐/预定项目}，订单号：{订单号}
——————————
日期：{日期}
时段：{午餐/晚餐/入住时段}
房间/桌位：{房间或包间}
人数：{人数}
当前状态：{待支付/待确认/待线下核销/已确认}
联系电话：{门店电话}
——————————
如需调整，请及时联系我们。
```

## 扫码点餐已提交

```text
【停云山居】
{联系人}，您好！
您在{桌号/桌名}的点餐已提交，订单号：{订单号}。
菜品会按提交批次安排制作；如需继续加菜，可继续扫码进入订单详情。
本单请在用餐结束后统一结账。
联系电话：{门店电话}
```

## 整单结账提醒

```text
【停云山居】
{联系人}，您好！
您在{桌号/桌名}的订单金额为 ¥{结账金额}。
{散客请在订单详情完成微信支付。/会员订单请由门店完成线下核销。}
订单号：{订单号}
联系电话：{门店电话}
```
