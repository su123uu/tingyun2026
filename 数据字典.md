# 停云山居小程序数据字典 v1.0

> 制定日期：2026-06-01
> 当前用途：统一前端 mock 数据、本地存储和未来云数据库字段。
> 说明：本文件定义“业务字段”；数据库集合、索引、权限和并发策略将在接入云服务时另行补充。

## 一、通用约定

| 约定 | 说明 |
|------|------|
| ID | 使用字符串，例如 `room_chunyue`、`RES_20260601_xxx` |
| 金额 | 单位统一为人民币元，类型为 `number` |
| 时间 | 存储为 ISO 8601 字符串，例如 `2026-06-15T14:00:00+08:00` |
| 手机号 | 存储为字符串，避免数值转换 |
| 可选字段 | 无值时使用空字符串、空数组或 `null`，不使用中文占位文字 |
| 状态值 | 使用稳定的英文枚举值，页面根据枚举值展示中文 |

## 二、用户与会员

### 2.1 用户 `users`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `user_id` | string | 是 | 小程序用户 ID |
| `openid` | string | 云端必填 | 微信用户标识；mock 阶段可为空 |
| `mobile` | string | 否 | 微信授权手机号 |
| `nickname` | string | 否 | 微信昵称或用户填写名称 |
| `avatar_url` | string | 否 | 头像地址 |
| `member_id` | string | 否 | 已绑定会员档案 ID |
| `customer_type` | enum | 是 | `guest` 散客、`member` 会员 |
| `created_at` | datetime | 是 | 首次进入时间 |

### 2.2 会员档案 `members`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `member_id` | string | 是 | 会员档案 ID |
| `mobile` | string | 是 | 用于匹配微信授权手机号 |
| `member_name` | string | 是 | 会员姓名 |
| `member_level` | string | 是 | 会员等级 |
| `member_status` | enum | 是 | `active`、`expired`、`disabled` |
| `benefit_start_at` | datetime | 否 | 权益开始时间 |
| `benefit_end_at` | datetime | 否 | 权益结束时间 |
| `points_balance` | number | 否 | 预留字段，本期用户端不展示 |

## 三、扫码点餐

### 3.1 菜品 `menu_items`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `item_id` | string | 是 | 菜品 ID |
| `category_id` | string | 是 | 菜品分类 ID |
| `item_type` | enum | 是 | `package` 套餐、`single` 单点 |
| `name` | string | 是 | 菜品名称 |
| `description` | string | 否 | 菜品描述 |
| `price` | number | 上架时必填 | 当前售价 |
| `image_url` | string | 否 | 图片地址 |
| `is_available` | boolean | 是 | 是否可加入购物车 |
| `sort_order` | number | 是 | 展示顺序 |

### 3.2 桌台会话 `table_sessions`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `session_id` | string | 是 | 桌台用餐会话 ID |
| `table_id` | string | 是 | 桌号 |
| `people_count` | number | 是 | 用餐人数 |
| `created_at` | datetime | 是 | 创建时间 |
| `expires_at` | datetime | 是 | 会话过期时间 |

### 3.3 点餐订单 `meal_orders`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `order_id` | string | 是 | 点餐订单 ID |
| `parent_order_id` | string | 否 | 加菜子订单关联的首单 ID |
| `table_session_id` | string | 是 | 桌台会话 ID |
| `table_id` | string | 是 | 桌号快照 |
| `people_count` | number | 是 | 用餐人数快照 |
| `user_id` | string | 是 | 下单用户 |
| `customer_type` | enum | 是 | `guest`、`member` |
| `items` | array | 是 | 下单时菜品快照 |
| `amount` | number | 是 | 本笔订单金额 |
| `kitchen_status` | enum | 是 | 厨房状态，见 3.4 |
| `settlement_status` | enum | 是 | 结算状态，见 3.5 |
| `points_amount` | number | 是 | 预留：线下积分扣款金额 |
| `wechat_pay_amount` | number | 是 | 微信支付金额 |
| `created_at` | datetime | 是 | 下单时间 |

### 3.4 厨房状态 `kitchen_status`

| 值 | 含义 |
|------|------|
| `pending_notice` | 已提交，正在通知厨房 |
| `kitchen_notified` | 厨房已收到 |
| `preparing` | 制作中 |
| `completed` | 已完成 |

### 3.5 结算状态 `settlement_status`

| 值 | 含义 |
|------|------|
| `pending_wechat_pay` | 散客待微信支付 |
| `wechat_paid` | 已微信支付 |
| `pending_offline_points` | 会员待线下积分扣款 |
| `points_deducted` | 已在线下积分程序扣款 |
| `pending_supplement` | 积分不足，待补差额 |
| `settled` | 已结清 |
| `refunding` | 退款处理中 |
| `refunded` | 已退款 |

### 4.3 预订订单 `reservation_orders`

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `order_id` | string | 是 | 预订订单 ID |

| `customer_type` | enum | 是 | `guest`、`member` |
| `room_ids` | array | 是 | 已选房间或包间 |
| `start_at` | datetime | 是 | 开始时间 |
| `end_at` | datetime | 是 | 结束时间 |
| `people_count` | number | 是 | 人数 |
| `meal_standard_id` | string | 否 | 用餐预订餐标 |
| `amount` | number | 是 | 应付总价 |
| `reservation_status` | enum | 是 | 预订状态，见 4.4 |
| `settlement_status` | enum | 是 | 结算状态，见 3.5 |

| `contact_name` | string | 是 | 联系人 |

| `created_at` | datetime | 是 | 创建时间 |

### 4.4 预订状态 `reservation_status`

| 值 | 含义 |
|------|------|
| `pending_payment` | 散客已临时锁定，待微信支付 |
| `paid_pending_confirmation` | 散客已支付，待确认 |
| `pending_confirmation` | 会员已提交，待人工确认 |
| `confirmed` | 已确认接待 |
| `rejected` | 无法接待，已拒绝 |
| `refunding` | 客服退款处理中 |
| `refunded` | 已退款 |
| `completed` | 已完成服务 |
| `cancelled` | 客服已取消 |
