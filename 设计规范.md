# 停云山居小程序 · 设计规范 v2.2

> 更新日期：2026-05-29  
> 适用范围：HTML 原型 + 后续微信小程序落地  
> 当前阶段：先打磨 HTML 前端页面，后端、腾讯云开发、支付、云打印暂未接入

---

## 一、设计定位

v1.0 同时承担两个核心目标：到店客人的扫码点餐，以及新客对停云山居的品牌理解。HTML 原型阶段先解决视觉、页面结构、假数据字段和交互节奏，确认后再迁移到微信小程序。

| 维度 | 规范 |
|------|------|
| 风格关键词 | 典雅、安静、留白、东方、克制 |
| 视觉气质 | 山间生活空间，不做强促销和连锁茶饮感 |
| 首页节奏 | 借鉴成熟小程序的信息效率，但不直接复刻具体品牌 |
| 页面密度 | 首屏只保留关键动作，模块之间有呼吸感 |
| 实现顺序 | HTML 原型先行，确认后迁移至小程序 WXML/WXSS |

首页应让用户快速理解三件事：这里能点餐、能预约、值得了解。首页不放会员信息卡，只保留会员活动入口。

---

## 二、v1.0 页面结构

底部 Tab 固定为 5 个：

| Tab | 页面 | 角色 |
|-----|------|------|
| 首页 | `home.html` / `pages/home/home` | 顶部轮播、点餐/预约入口、山居介绍、会员活动入口 |
| 点餐 | `menu.html` / `pages/menu/menu` | 扫码点餐、购物车、模拟支付 |
| 介绍 | `intro.html` / `pages/intro/intro` | 山居介绍、空间/住宿/餐厅展示、联系方式 |
| 订单 | `orders.html` / `pages/orders/orders` | 假数据订单列表、状态筛选、详情入口 |
| 我的 | `profile.html` / `pages/profile/profile` | 会员信息、历史消费、权益、客服入口 |

预约预订在 v1.0 先做展示/占位入口，可展示住宿、用餐、空间预订类型，但不做真实库存、日历锁房和在线支付。

## 三、色彩系统

| 名称 | 色值 | 用途 |
|------|------|------|
| 山居绿 | `#4F6F52` | 主按钮、点餐主流程、导航选中、品牌强调 |
| 古铜棕 | `#8B7355` | 预约预订模块，用于和点餐区分 |
| 宣纸白 | `#F8F6F1` | 页面底色 |
| 暖米色 | `#E8E4DC` | 卡片底、图片占位、浅色分隔 |
| 装饰金 | `#C9A962` | 会员、细线、徽章、小面积强调 |
| 墨黑 | `#1A1A1A` | 主标题、价格、关键数字 |
| 墨灰 | `#4A4A4A` | 正文、说明文字 |
| 雾灰 | `#7C8A8C` | 辅助说明、空状态、次级图标 |
| 禁用灰 | `#BBBBBB` | 不可点击、未完成状态 |

色彩原则：

- 金色只做细线、徽章、少量会员强调，不做大面积背景。
- 点餐流程以山居绿为主，预约模块以古铜棕为辅。
- 页面背景保持浅暖，不使用冷蓝、亮紫、强商业渐变。
- 深色会员卡可以保留，但必须克制，避免变成促销广告。

## 四、字体与排版

HTML 原型可使用系统字体或 Noto Serif SC 作为品牌标题参考；微信小程序落地以系统字体为主，通过字重、间距和留白体现东方感。

| 用途 | HTML 建议 | 小程序建议 | 说明 |
|------|-----------|------------|------|
| 页面标题 | 22-28px | 40-48rpx | 首页品牌名、详情页标题 |
| 区块标题 | 16-18px | 30-34rpx | 点餐分类、活动、会员权益 |
| 卡片标题 | 15-17px | 28-32rpx | 商品名、活动名、房间名 |
| 正文 | 13-15px | 24-28rpx | 说明、介绍、菜品描述 |
| 辅助文字 | 11-13px | 20-24rpx | 时间、状态、提示、标签 |
| 价格/数字 | 18-24px | 30-42rpx | 订单金额、会员消费额 |

排版原则：

- 手机页面左右安全边距建议 16px / 32rpx。
- 首页模块间距建议 16-24px / 32-48rpx。
- 卡片内部标题和正文间距控制在 6-10px / 12-20rpx。
- 不使用 emoji 作为正式图标，HTML 原型也尽量使用 SVG 线性图标。

## 五、首页布局规范

首页目标是“快速点餐、知道可预约、感受到山居、看到会员活动”，但不追求满屏功能。

推荐结构：

| 顺序 | 模块 | 规范 |
|------|------|------|
| 1 | 顶部轮播 | 3 张以内，宽高比 `3:4`，顶到页面上方，主视觉清爽 |
| 2 | 快捷入口 | 点餐、预约两个同尺寸入口，放在同一张白色功能卡中 |
| 3 | 山居介绍 | 一张图 + 一个主标题，点击进入介绍页 |
| 4 | 会员活动 | 一张图 + 一个主标题，点击进入活动页 |

首页尺寸建议：

- 顶部轮播宽高比固定为 `3:4`，无外边距、无圆角。
- 快捷入口功能卡左右边距约 14px，上压轮播约 50px。
- 点餐/预约入口左右等宽，中间一条细分隔线。
- 山居介绍、会员活动卡左右边距与功能卡一致，卡片圆角约 14px。
- 山居介绍和会员活动只保留“图片 + 主标题”，不放副标题和正文。

## 六、核心组件规范

### 底部导航

- 5 个 Tab：首页、点餐、介绍、订单、我的。
- 选中态使用山居绿，不用金色做主选中态。
- 背景使用半透明白或宣纸白，顶部细分隔线。

### 卡片

- 用于商品、订单、活动、会员权益等独立信息块。
- 背景默认白色或半透明白，少量使用暖米色。
- 阴影轻，不做厚重悬浮感。

```css
box-shadow: 0 8px 24px rgba(26, 26, 26, 0.045);
```

### 按钮

- 主按钮：山居绿底白字。
- 预约按钮：古铜棕底白字。
- 次级按钮：白底描边或浅米色底。
- 禁用按钮：禁用灰或低透明度。

### 状态标签

| 状态 | 颜色 |
|------|------|
| 待支付/待处理 | 装饰金 |
| 已支付/制作中 | 山居绿 |
| 已完成 | 雾灰 |
| 已取消 | 禁用灰 |

## 七、假数据阶段规范

当前没有后端接口，腾讯云开发未购买，所有页面先使用前端假数据实现完整体验。

开发规则：

- HTML 阶段允许把假数据写在页面脚本中，但字段命名要接近未来接口。
- 页面交互要像真实业务一样完整，例如加购、合计、订单状态、会员等级、历史消费。
- 涉及后端能力的按钮显示真实意图，但给出明确状态反馈。
- 不把“开发中”作为主要页面内容，预约这类 v1.0 占位模块也要有正式展示。

暂不接入但需要预留的能力：

| 能力 | 当前表现 | 未来接入 |
|------|----------|----------|
| 微信支付 | 前端模拟支付成功/失败 | 云函数 + 微信支付 |
| 云数据库 | 本地假数据/页面状态 | 云开发数据库 |
| 会员注册 | 模拟授权和会员资料 | 微信手机号授权 + 用户表 |
| 厨房打印 | 下单成功展示“已通知厨房” | 云打印机 API |
| 商家通知 | 前端展示通知状态 | 飞书/微信通知 |
| 预约库存 | 仅展示和客服引导 | 房态/桌态/时段管理 |

## 八、业务页设计规则

### 点餐页

- 顶部显示桌号，如果没有扫码参数，则显示“堂食点餐”或模拟桌号。
- 分类保持清晰：套餐、凉菜、热菜、主食、饮品。
- 商品卡片包含名称、描述、价格、加减数量。
- 底部固定购物车栏，显示数量、合计、结算按钮。
- 模拟支付成功后生成假订单，并跳转订单页或订单详情页。

### 介绍页

- 重点展示山居气质和空间信息，不要做成普通企业介绍。
- 内容包括：关于山居、禅修、茶艺、餐饮、住宿、房间/空间列表、联系方式。
- 房间、餐厅、空间资料使用现有对照表假数据。

### 订单页

- 显示点餐订单为主，预约订单可作为占位或示例。
- 支持状态筛选：全部、待处理、制作中、已完成、已取消。
- 订单卡片包含订单号、时间、桌号、商品摘要、金额、状态。
- 详情页展示商品明细、支付状态、厨房通知/打印状态占位。

### 我的页

- 会员信息是真实字段展示，即使当前是假数据。
- 字段包括：会员等级、折扣、累计消费、历史订单数、活动权益、生日礼遇等。
- 未登录状态展示“成为会员”；模拟登录后展示会员卡和权益。

### 活动详情

- v1.0 不做在线报名表。
- 活动详情展示时间、地点、内容、适合人群。
- 报名按钮使用“电话咨询/联系客服/添加微信”方向。

## 九、验收标准

HTML 原型阶段应满足：

- 首页不拥挤，轮播、点餐/预约、山居介绍、活动模块视觉重量均衡。
- 5 个 Tab 都不是空白页，预约入口也有正式展示。
- 点餐页可完成从加购、合计、模拟支付到订单展示的闭环。
- 会员页展示等级、折扣、历史消费、活动权益等真实字段。
- 所有后端未接入能力都有清晰占位，不影响用户理解。
- 视觉统一使用本规范色彩、圆角、间距和状态规则。

---

*本规范以 HTML 原型为当前主设计稿，后续迁移到微信小程序时保持页面结构、字段和交互逻辑一致。*
