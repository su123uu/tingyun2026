# 停云山居小程序验收测试清单 v1.0

> 当前清单用于前端 mock 阶段。真实云服务、支付、退款、打印和后台操作需要在接入后补充联调测试。

## 一、用户身份

- [ ] 未授权手机号时按散客展示。
- [ ] 授权已录入的会员手机号后切换为会员。
- [ ] 授权未录入手机号时仍按散客展示。
- [ ] 页面不展示积分余额、线下账户余额、积分明细和积分抵扣记录。

## 二、扫码点餐

- [ ] 未扫码时允许浏览菜单和加入购物车。
- [ ] 未扫码时提交订单会提示先扫描桌码。
- [ ] 扫描合法桌码后保存桌号和用餐人数。
- [ ] 散客提交订单后进入待微信支付，模拟支付成功后通知厨房。
- [ ] 会员提交订单后直接通知厨房，订单显示待线下会员账户核对。
- [ ] 同一桌台会话再次下单时生成追加菜子订单，订单列表仍只显示一条首单。
- [ ] 点餐详情展示首单、追加菜和总金额。

## 三、用餐预订

- [ ] 少于 6 人不可预订。
- [ ] 填写日期、时段和人数后展示可用包间。
- [ ] `6-12` 人最多选择 1 个包间。
- [ ] `13-22` 人最多选择 2 个包间，`23` 人及以上可选择全部包间。
- [ ] 所选包间总容量不足时不可提交。
- [ ] 餐标价格分别为 `40`、`168`、`218`、`318` 元/人。
- [ ] 总价等于人数乘以餐标单价。
- [ ] 散客提交后待支付状态不占用包间，支付成功后占用包间。
- [ ] 会员提交后占用包间并等待人工确认、线下会员账户核对。

## 四、住宿预订

- [ ] 离店日期必须晚于入住日期。
- [ ] 根据入住日期范围展示房态，已被占用的房间不可选择。
- [ ] 住宿预订记录人数但不按房间容量限制提交。
- [ ] 散客总价等于所选房间原价之和乘以晚数。
- [ ] 会员总价等于所选房间会员价之和乘以晚数。
- [ ] 散客提交后待支付状态不占用房间，支付成功后占用房间。
- [ ] 会员提交后占用房间并等待人工确认、线下会员账户核对或权益核销。
- [ ] 预订详情只提示联系客服取消，不显示退款按钮。

## 五、活动报名

- [ ] 公开免费活动允许散客和会员报名。
- [ ] 仅限会员活动拒绝散客报名。
- [ ] 收费公开活动允许散客报名并模拟微信支付。
- [ ] 收费活动会员按会员价等待线下会员账户核对。
- [ ] 单次报名最多 2 人。
- [ ] 报名人数不可超过剩余名额。
- [ ] 免费活动允许自行取消并释放名额。
- [ ] 收费活动不提供自行取消入口。

## 六、接入云服务后补测

- [ ] 微信手机号授权解密。
- [ ] 多用户同时预订时，未支付订单不占房，支付成功时做最终房态校验。
- [ ] 散客未支付、支付取消或支付失败时，房间保持可选。
- [ ] 微信支付回调、重复回调和退款。
- [ ] 厨房通知、云打印和失败重试。
- [ ] 后台人工确认、拒绝、取消、会员账户核对和权益核销登记。
